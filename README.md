# Healthcare Capstone Project: Predicting 30-Day Hospital Patient Readmissions

**SFHA Advanced Data + AI Program – Week 8 Capstone**

---

## Project Overview

This project builds an end-to-end machine-learning pipeline to predict whether
a hospital patient will be readmitted within 30 days of discharge.  It
demonstrates all three required core concepts of the SFHA Advanced Data + AI
Program capstone:

| Core Concept | Implementation |
|---|---|
| **Data Processing** | Cleansing, feature engineering, 5 visualisations |
| **Machine Learning** | Random Forest & Logistic Regression classification |
| **Generative AI** | ChatGPT ideation, GitHub Copilot assistance, LLM prompt system |

---

## Problem Statement

Hospital readmissions affect 15–20% of discharged patients and cost the
healthcare system billions of dollars annually.  By predicting high-risk
patients at discharge, care teams can intervene proactively—reducing
readmissions, improving patient outcomes, and lowering costs.

**Who benefits:** patients, care coordinators, hospital administrators, and
health system payers.

---

## Dataset

A synthetic dataset of **1,000 patients** is generated by
`data/generate_synthetic_data.py` and saved as
`data/hospital_readmissions.csv`.

| Column | Description |
|--------|-------------|
| `patient_id` | Unique patient identifier |
| `age` | Patient age (18–95) |
| `gender` | Male / Female |
| `admission_type` | Emergency / Urgent / Elective |
| `diagnosis_category` | Primary diagnosis group |
| `num_lab_procedures` | Number of lab tests performed |
| `num_medications` | Number of medications prescribed |
| `num_previous_admissions` | Prior hospital stays in past year |
| `length_of_stay` | Days in hospital (current visit) |
| `discharge_disposition` | Destination after discharge |
| `has_diabetes` | Comorbidity flag (0/1) |
| `has_hypertension` | Comorbidity flag (0/1) |
| `readmitted_within_30_days` | **Target variable** (0 = No, 1 = Yes) |

## Results Summary

| Model | Accuracy | F1 Score | ROC-AUC |
|-------|----------|----------|---------|
| Random Forest | ~74% | ~0.49 | ~0.79 |
| Logistic Regression | ~70% | ~0.54 | ~0.73 |

**Best model:** Random Forest (highest ROC-AUC)  
**Top predictors:** admission_type_enc, has_diabetes, num_previous_admissions,
num_medications, polypharmacy

> Note: The synthetic dataset has a ~27% readmission rate—slightly above the
> published 15–20%—because the logistic risk model uses strong coefficients
> to produce meaningful predictive signal from 1,000 samples.  The relative
> risk ordering between patient groups (Emergency > Urgent > Elective;
> diabetic > non-diabetic) correctly mirrors clinical literature.

## Core Concept Locations

### Data Processing *(Part 1 of `notebooks/capstone_analysis.py`)*
- **Cleansing** [§1.2]: duplicate removal, missing-value imputation, outlier clamping
- **Manipulation** [§1.3]: 4 engineered features, label encoding
- **Visualisation** [§1.4]: 5 publication-quality plots saved to `outputs/`

### Machine Learning *(Part 2 of `notebooks/capstone_analysis.py`)*
- **Classification** [§2.2]: Random Forest + Logistic Regression
- **Evaluation** [§2.3]: accuracy, precision, recall, F1, ROC-AUC
- **Visuals** [§2.4–2.6]: confusion matrices, ROC curves, feature importance

### Generative AI *(Part 3 of `notebooks/capstone_analysis.py`)*
- **Ideation** [§3.1]: ChatGPT GPT-4o prompted for feature suggestions
- **Coding assistance** [§3.1]: GitHub Copilot used throughout development
- **System integration** [§3.2–3.4]: `build_patient_risk_prompt()` generates
  LLM-ready prompts for patient risk summaries; `generate_analysis_report()`
  produces a structured Markdown report from ML outputs
